# 扩展需求描述引导

帮助你描述对现有功能的扩展方向。

## 请回答以下问题

### 1. 基于哪个现有功能扩展？
> 要扩展的 skill 名称或功能模块

### 2. 为什么需要扩展？
> 当前功能的不足之处

### 3. 扩展后的能力是什么？
> 新增的具体能力

### 4. 是否影响现有行为？
> 向后兼容？需要迁移？

### 5. 扩展的范围有多大？
> 改动量预估：小（几个函数）/ 中（新增模块）/ 大（架构调整）

### 6. 是否需要新的依赖？
> 新的库、服务、外部 API

---

## 输出模板

```
## 扩展: {skill名称} - {扩展方向}

**基础**: {现有功能}
**原因**: {扩展动机}
**新增能力**: {具体能力}
**兼容性**: {向后兼容/需迁移}
**范围**: {小/中/大}
**新依赖**: {依赖列表}
```
